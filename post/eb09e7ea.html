孤鸿API开放平台

制作这个项目主要是为了给开发者提供便捷的接口服务，同时我也会为实验室的小伙伴提供几个在实验中经常用到的算法API，方便大家做实验。这个文章记录项目制作过程中遇到得到问题及重点知识的笔记。

首先这个项目已经在本地成功运行，前端还是有点小bug（不影响使用），但是项目部署到服务器上后，后端一直无法获取用户的登录态，导致用户一点击调用接口就跳转到了登录页，折腾了半天没搞好，而且项目的前端不是我写的，是别人的用react写的，我不是很懂，实在是难以调试，打算日后用空我重新用vue写一下项目的前端。

项目的整个后端主要由我一个人完成，一共分为5个模块：后端主模块（用于接收前端页面请求，实现用户注册，用户登录，管理员管理接口等主要功能），客户端SDK模块，API网关模块，公共模块、实际接口模块。下面具体介绍项目的流程。

后端主项目相当于一个后台管理系统的模板，对数据库实现增删改查的功能。数据库一共用三张表，一个用户表，用户表中包含用户是否是管理员还是普通用户的信息；一个接口信息表，还有一个用户接口信息表，主要是记录用户对接口的调用次数信息。完成前端和后端主项目的初始化后，为了让流程跑通，我先是开发了一个模拟接口的项目guhong-interface。它运行的端口为8123。模拟接口写完后我们就该考虑如何调用它。

API 接口调用方式

  1. HttpClient
  2. RestTemplate
  3. 第三方库（OKHttp、Hutool）

我选择的是Hutool工具来调用接口。

同时为了保证调用接口的安全性，我们引入了API签名认证。

API签名认证

（抛出问题）

为什么需要API签名认证？

  1. 保证安全性，不能随便是条狗就来调用
  2. 适用于无需保存登录态的场景。只认签名，不关注用户登录态。

什么是重放攻击？

重复攻击破坏认证的正确性。例如攻击者拦截到签名认证，之后再把签名认证发送给服务器，让服务器以为攻击者是你本人。

（答案）

本质：

  1. 签发签名
  2. 校验签名

实现：

通过 http request header 头传递参数

  • 参数1：accessKey 调用的标识（复杂、无序、无规律）

  • 参数2：secretKey 密钥（复杂、无序、无规律）==该参数不能放到请求头中==

  • 参数3：body 用户请求参数

  • 参数4：sign 签名（”不可解密的值”）

  • 参数5：nonce 随机数，只能用一次，服务端要保存用过的随机数

  • 参数6：timestamp 时间戳，校验时间戳是否过期。

==ak、sk类似于用户名和密码，不同点在于ak、sk是没有登录状态的。==

注意点：

密钥不能在服务器中以明文方式发送，有可能被拦截，所以要加密。

加密方式：对称加密、非对称加密、md5 签名（单向加密不可解密）

用户参数 + 密钥 => 签名生成算法（MD5、HMac、Sha1） => 不可解密的值

服务端怎么知道这个 “不可解密的值” 对不对？

服务端从数据库中查出secretKey，根据客户端传来body和secretKey生成签名，如果签名和客户端传来的一致，就表示一致。

防止重放攻击：

  • 参数5：nonce 随机数，只能用一次，服务端要保存用过的随机数

  • 参数6：timestamp 时间戳，校验时间戳是否过期。

==API签名认证是一个很灵活的设计，具体要有哪些参数看实际业务需求（比如userId、appId、version、固定值）。==

SDK开发

这个项目就是一个API中间商，对接 模拟接口 供用户调用，但是当用户来调用我们的接口也不能让他写那些与调用接口无关的代码。（总不能让用户每次调用接口都自己写一个签名算法吧）

理想情况：用户只需要关心调用哪些接口、传递哪些参数，就跟调用自己写的代码一样简单。

开发 starter

开发 starter 的好处：用户引入之后，可以直接在 application.yml 中写配置，自动创建客户端。 需要在porn文件引入依赖。

Starter 开发流程：

  1. 引入spring-boot-configuration-processor，作用：在yml配置文件中生成提示代码，移除build。

  1. 删除生成的启动类，创建一个类，使用@ConfigurationProperties读取配置文件中的配置。

    这个 GhApiClient里面就封装了调用接口的准备工作。

  2. 在resource目录下新建META-INF文件夹，文件夹下新建spring.factories文件，文件内容指向配置类。

  3. install: 使用Maven的install命令打包安装至本地仓库。

然后就可以在别的项目里引用这个jar包。

  1. package只是将项目打包成jar或war文件，不会将其安装到本地仓库中。
  2. install会将项目打包成jar或war文件，并将其安装到本地仓库中，以便其他项目可以引用它。
